Installation

Supported language:

  • Python: 3.7

Note

We highly recommend installing and using the free Anaconda distribution of Python (or Miniconda, if you don’t want all of the extra packages that come built-in with Anaconda), which works on Mac, Linux, and Windows, both on normal computers and institutional clusters and doesn’t require root permissions.

wrf_pipeline_tools depends on the following libraries

  • argparse

  • matplotlib

  • numpy

  • python=3.7

  • pip

  • tqdm

  • xarray

  • metpy

  • proj

  • geos

  • pop-tools

  • cartopy

  • netCDF4

  • xesmf

After the required packages are installed, wrf_pipeline_tools can be installed from source.

Clone from Github

You can obtain it directly from the Github repo

git clone https://www.github.com/bnb32/wrf_pipeline_tools.git

Edit environment variables in pipeline_env_info.sh

You need to have the following nc files in an Invariant_Data directory:
/gpfs/fs1/p/cesmdata/cseg/ncl/map_gx1v6_to_fv1.9x2.5_bilin_da_090296.nc
/gpfs/fs1/p/cesmdata/cseg/inputdata/lnd/clm2/griddata/fracdata_1.9x2.5_gx1v6_c090206.nc
/gpfs/fs1/p/cesmdata/cseg/inputdata/atm/cam/topo/USGS-gtopo30_1.9x2.5_remap_c050602.nc

You also need to have WRF and WPS directories in the directory defined byDIR_HOME.
